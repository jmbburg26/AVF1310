      Weekly Research

      Week One

      • 1. Twitter is already quite popular for signing in to websites and apps. There are several blogs that I visit that I use my Twitter information to log in and add comments to posts. The Website Magazine has a good article about the benefits of using social for login. The Benefits of Social Login by Allison Howen. By using Twitter for our login, it allows us to pre-fill in the account information and not have to remember another username and password. It helps with our online security and gives us a better experience on the web. Payments are an interesting way to use Twitter. If Twitter were set up with a payment system such as Google Wallet or Paypal, it would allow for person-to-person payments via direct message or payments for goods at a store.

      • 2. JSONP makes some things immensely easier for developers. JSONP helps the developer to access additional sites and information on the web without having to hard code it into their app or site. For our apps, JSONP allows us to pull in data using different APIs. We can bring the data to the user instead of the user having to go looking for it.

      • 3. One resource that would be useful in my app is one of the many calendar APIs. The My School Holidays API would be useful for my homework application because it allows the app to automatically add school holidays to the user’s calendar. Syncing between different accounts like Google and iCloud would allow for better cross device tracking.

      • 4. I would like to create a better interface for my app. I am thinking about going with a tiled interface that has a slight Windows 8 feel to it. I would like to add some Facebook feed integration. Having the feed from the MDVBS Facebook group or the Full Sail feed from the Schools App would be extremely helpful and would allow the user to ask question and interact with fellow students. I would also like to get Google Calendar syncing to work when adding an assignment within the app.

      • 5. My biggest lesson learned so far has been to slow down. I had a few times where I got behind on a project and didn’t get it completely finished or not turned in on time. The next day when I wasn’t stressed out and pushed against a deadline, I was able to work through things and figure out the problems I had with my code. Not getting worked up and going through the process of proper debugging is the key to success for me.

      Week Two

      • 1. Due to the fact that my app is a list-based app, I believe that it would not be too difficult to add the proper accessibility to my app. For the visually impaired, I could change the interface to have larger buttons that would be easier to press and have the proper tags for the screen readers to parse.

      • 2. I found some videos on YouTube about screen readers and found that often times it is hard for the user to be able to get the scope or structure of the document. They cannot see the whole text at once and have to read it one word at a time. This would create problems if things must be read in a specific sequence.

        One advantage I think they have is the speed in which they can ingest information. One of the videos I watched had a man listening to Screen Reader at 119 words per minute. I can’t read that fast and it’s amazing that people can do that.

      • 3. Motorola Moto X- The Moto X has been designed to be able to be controlled almost completely via voice commands. It has deep integration with Google Now and the user is able to send texts, set reminders, etc.

        Apple iPad- The iPad is considered to be one of the best devices for accessibility due to Voice Over, large app icon interface, and a large screen combine for a very accessible device.

      • 4. Mobile Accessibility is an app for Android that is designed to allow better use for blind and low vision users.

        iOS 7 has added many new features for better accessibility. Things like Voice Over, Braille, and enhancements in Siri attempt to make things easier for those with poor vision and hearing.
